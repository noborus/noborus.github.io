E.322. リリース7.3.11

<title>Release 7.3.11</title>

<title>Release date:</title> リリース日: 2005-10-04

This release contains a variety of fixes from 7.3.10. このリリースは7.3.10の各種不具合を修正したものです。

E.322.1. バージョン7.3.11への移行

<title>Migration to Version 7.3.11</title>

A dump/restore is not required for those running 7.3.X. However, if you are upgrading from a version earlier than 7.3.10, see <xref linkend="release-7-3-10"/>. 7.3.Xからの移行ではダンプ/リストアは不要です。 しかしながら、7.3.10より以前のバージョンからアップグレードする場合は、E.323を参照してください。

E.322.2. 変更点

<title>Changes</title>
<listitem><para>Fix error that allowed <command>VACUUM</command> to remove <literal>ctid</literal> chains too soon, and add more checking in code that follows <literal>ctid</literal> links</para>

  • VACUUMによってctidチェーンがすぐに削除されてしまうエラーを修正し、ctidリンクを追跡するコードにおいてより多くの検査を行うようにしました。

    <para>This fixes a long-standing problem that could cause crashes in very rare circumstances.</para></listitem>

    これにより、長い間存在した、非常に稀な環境におけるクラッシュという問題が修正されました。

    • <listitem><para>Fix <type>CHAR()</type> to properly pad spaces to the specified length when using a multiple-byte character set (Yoshiyuki Asaba)</para>

  • マルチバイト文字セットを使用する場合に、CHAR()の空白文字埋め込みが適切に指定した長さになるように修正しました。(Yoshiyuki Asaba)

    <para>In prior releases, the padding of <type>CHAR()</type> was incorrect because it only padded to the specified number of bytes without considering how many characters were stored.</para></listitem>

    以前のリリースでは、CHAR()の空白文字埋め込みが正しくありませんでした。 格納する文字数を考慮せずに、単に指定されたバイト数で空白文字を埋め込んでいただけでした。

    • <listitem><para>Fix missing rows in queries like <literal>UPDATE a=... WHERE a...</literal> with GiST index on column <literal>a</literal></para></listitem>

  • a列にGiSTインデックスを持つ場合のUPDATE a=... WHERE a...といった問い合わせにおいて行が見つからない点を修正しました。

    • <listitem><para>Improve checking for partially-written WAL pages</para></listitem>

  • 部分的に書き込まれたWALページに対する検査を改良しました。

    • <listitem><para>Improve robustness of signal handling when SSL is enabled</para></listitem>

  • SSL有効時のシグナルの扱いに対する堅牢性を向上しました。

    • <listitem><para>Various memory leakage fixes</para></listitem>

  • さまざまなメモリリークを修正しました。

    • <listitem><para>Various portability improvements</para></listitem>

  • 移植性の向上をいろいろと行いました。

    • <listitem><para>Fix PL/pgSQL to handle <literal>var := var</literal> correctly when the variable is of pass-by-reference type</para></listitem>

  • PL/pgSQLにおいて、変数が参照渡し型の場合のvar := varの扱いを修正しました。
