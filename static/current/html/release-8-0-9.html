E.271. リリース8.0.9

<title>Release 8.0.9</title>

<title>Release date:</title> リリース日: 2006-10-16

This release contains a variety of fixes from 8.0.8. For information about new features in the 8.0 major release, see <xref linkend="release-8-0"/>. このリリースは8.0.8の各種不具合を修正したものです。 8.0メジャーリリースにおける新機能についてはE.280を参照してください。

E.271.1. バージョン8.0.9への移行

<title>Migration to Version 8.0.9</title>

A dump/restore is not required for those running 8.0.X. However, if you are upgrading from a version earlier than 8.0.6, see <xref linkend="release-8-0-6"/>. 8.0.Xからの移行ではダンプ/リストアは不要です。 しかしながら、8.0.6より前のバージョンからアップグレードする場合は、E.274を参照してください。

E.271.2. 変更点

<title>Changes</title>
<listitem><para>Fix crash when referencing <literal>NEW</literal> row values in rule WHERE expressions (Tom)</para></listitem>

  • ルールのWHERE式でNEW行の値を参照する時のクラッシュを修正しました。(Tom)

    • <listitem><para>Fix core dump when an untyped literal is taken as ANYARRAY</para></listitem>

  • 型指定がないリテラルがANYARRAYと解釈される場合のコアダンプを修正しました。

    • <listitem><para>Fix mishandling of AFTER triggers when query contains a SQL function returning multiple rows (Tom)</para></listitem>

  • 複数行を返すSQL関数を含む問い合わせにおけるAFTERトリガの間違った扱いを修正しました。(Tom)

    • <listitem><para>Fix <command>ALTER TABLE ... TYPE</command> to recheck <literal>NOT NULL</literal> for <literal>USING</literal> clause (Tom)</para></listitem>

  • ALTER TABLE ... TYPEがUSING 句に対するNOT NULLを再検査するように修正しました。(Tom)

    • <listitem><para>Fix <function>string_to_array()</function> to handle overlapping matches for the separator string</para>

  • 区切り文字と重なって一致する文字を扱うようstring_to_array()を修正しました。

    <para>For example, <literal>string_to_array('123xx456xxx789', 'xx')</literal>. </para></listitem>

    例えばstring_to_array('123xx456xxx789', 'xx')です。

    • <listitem><para>Fix corner cases in pattern matching for <application>psql</application>'s <literal>\d</literal> commands</para></listitem>

  • psqlの\dコマンドに対するパターンマッチにおけるちょっとした部分を修正しました。

    • <listitem><para>Fix index-corrupting bugs in /contrib/ltree (Teodor)</para></listitem>

  • /contrib/ltreeにおけるインデックス破損不具合を修正しました。(Teodor)

    • <listitem><para>Numerous robustness fixes in <application>ecpg</application> (Joachim Wieland)</para></listitem>

  • ecpgをより強固にするためいくつかの修正をしました。

    • <listitem><para>Fix backslash escaping in /contrib/dbmirror</para></listitem>

  • /contrib/dbmirrorにおけるバックスラッシュエスケープ処理を修正しました。

    • <listitem><para>Fix instability of statistics collection on Win32 (Tom, Andrew)</para></listitem>

  • Win32における統計情報収集の不安定性を修正しました。(Tom, Andrew)

    • <listitem><para>Fixes for <systemitem class="osname">AIX</systemitem> and <productname>Intel</productname> compilers (Tom)</para></listitem>

  • AIXおよびIntelコンパイラに対して修正を行いました。(Tom)
