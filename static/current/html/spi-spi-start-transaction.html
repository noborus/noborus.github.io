SPI_start_transaction

SPI_start_transaction <refpurpose>start a new transaction</refpurpose> — 新たなトランザクションを開始します

概要

void SPI_start_transaction(void)

説明

<title>Description</title>

<function>SPI_start_transaction</function> starts a new transaction. It can only be called after <function>SPI_commit</function> or <function>SPI_rollback</function>, as there is no transaction active at that point. Normally, when an SPI-using procedure is called, there is already a transaction active, so attempting to start another one before closing out the current one will result in an error. SPI_start_transactionは新たなトランザクションを開始します。 そこがトランザクション内では無いように、SPI_commitまたはSPI_rollbackの後でのみ呼び出しできます。 通常は、何らかSPIを使ったプロシージャが呼ばれたとき、トランザクションは既に開始済みです。そのため、現在のトランザクションを閉じる前に別のトランザクションを開始しようとするとエラーになります。

This function can only be executed if the SPI connection has been set as nonatomic in the call to <function>SPI_connect_ext</function>. この関数はSPI接続がSPI_connect_extの呼び出しで非原子的と設定されている場合のみ、実行できます。
