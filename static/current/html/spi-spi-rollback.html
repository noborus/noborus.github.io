SPI_rollback

SPI_rollback <refpurpose>abort the current transaction</refpurpose> — 現在のトランザクションを中断します。

概要

void SPI_rollback(void)

説明

<title>Description</title>

<function>SPI_rollback</function> rolls back the current transaction. It is approximately equivalent to running the SQL command <command>ROLLBACK</command>. After a transaction is rolled back, a new transaction has to be started using <function>SPI_start_transaction</function> before further database actions can be executed. SPI_rollbackは現在のトランザクションをロールバックします。 これはSQLコマンドのROLLBACKを実行することと概ね同等です。 トランザクションをロールバックした後、続くデータベース操作を実行する前にSPI_start_transactionを使用して新たなトランザクションを開始しなければなりません。

This function can only be executed if the SPI connection has been set as nonatomic in the call to <function>SPI_connect_ext</function>. この関数はSPI接続がSPI_connect_extの呼び出しで非原子的と設定されている場合のみ、実行できます。
