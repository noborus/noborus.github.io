E.287. リリース7.4.24

<title>Release 7.4.24</title>

<title>Release date:</title> リリース日: 2009-02-02

This release contains a variety of fixes from 7.4.23. For information about new features in the 7.4 major release, see <xref linkend="release-7-4"/>. このリリースは7.4.23に対し、各種の不具合を修正したものです。 7.4メジャーリリースにおける新機能についてはE.311を参照してください。

E.287.1. バージョン7.4.24への移行

<title>Migration to Version 7.4.24</title>

A dump/restore is not required for those running 7.4.X. However, if you are upgrading from a version earlier than 7.4.11, see <xref linkend="release-7-4-11"/>. 7.4.Xからの移行ではダンプ/リストアは不要です。 しかしながら、7.4.11より前のバージョンからアップグレードする場合は、E.300を参照してください。

E.287.2. 変更点

<title>Changes</title>

  • Improve handling of URLs in <function>headline()</function> function (Teodor) headline()関数におけるURLの扱いを改良しました。(Teodor)

  • Improve handling of overlength headlines in <function>headline()</function> function (Teodor) headline()関数における延長見出しの扱いを改良しました。(Teodor)

  • Prevent possible Assert failure or misconversion if an encoding conversion is created with the wrong conversion function for the specified pair of encodings (Tom, Heikki) 符号化方式の変換が符号化方式の組み合わせの指定を間違えた変換関数で作成された場合にあり得る、アサートの失敗や誤変換を防止します。(Tom, Heikki)

  • Avoid unnecessary locking of small tables in <command>VACUUM</command> VACUUMにおいて小規模なテーブルに対する不要なロックを防止しました。 (Heikki)

  • Fix uninitialized variables in <filename>contrib/tsearch2</filename>'s <function>get_covers()</function> function (Teodor) contrib/tsearch2のget_covers()関数における未初期化変数を修正しました。(Teodor)

  • Fix bug in <function>to_char()</function>'s handling of <literal>TH</literal> format codes (Andreas Scherbaum) to_char()のTH書式を扱うコードの不具合を修正しました。(Andreas Scherbaum)

  • Make all documentation reference <literal>pgsql-bugs</literal> and/or <literal>pgsql-hackers</literal> as appropriate, instead of the now-decommissioned <literal>pgsql-ports</literal> and <literal>pgsql-patches</literal> mailing lists (Tom) すべての文書において、現在活動していないpgsql-portsとpgsql-patchesメーリングリストの代わりに、pgsql-bugsまたはpgsql-hackersを参照するようにしました。
