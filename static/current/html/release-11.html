E.6.1. 概要

<title>Overview</title>

Major enhancements in <productname>PostgreSQL</productname> 11 include: PostgreSQL 11の主な強化点には以下のものがあります。

Items in this list summarize one or more items below

  • Improvements to partitioning functionality, including: パーティションニング機能の改善:

    • Add support for partitioning by a hash key ハッシュキーによるパーティションに対応しました。

    • Add support for <literal>PRIMARY KEY</literal>, <literal>FOREIGN KEY</literal>, indexes, and triggers on partitioned tables パーティションテーブルに対するPRIMARY KEY、FOREIGN KEY、インデックス、トリガに対応しました。

    • Allow creation of a <quote>default</quote> partition for storing data that does not match any of the remaining partitions 既存パーティションのどれにも一致しないデータを格納する「デフォルト」パーティションを作れるようにしました。

    • <command>UPDATE</command> statements that change a partition key column now cause affected rows to be moved to the appropriate partitions パーティションキー列を変更するUPDATE文では、変更行が適切なパーティションに移動するようになりました。

    • Improve <command>SELECT</command> performance through enhanced partition elimination strategies during query planning and execution 問い合わせのプラン作成時と実行時の強化されたパーティション排除方式によってSELECT性能を改善しました。

  • Improvements to parallelism, including: 並列処理の改善:

    • <command>CREATE INDEX</command> can now use parallel processing while building a B-tree index CREATE INDEXがBツリーインデックスを構築するときに並列処理できるようになりました。

    • Parallelization is now possible in <command>CREATE TABLE ... AS</command>, <command>CREATE MATERIALIZED VIEW</command>, and certain queries using <literal>UNION</literal> 並列化がCREATE TABLE ... AS、CREATE MATERIALIZED VIEW、および、UNIONを使用するいくらかの問い合わせで可能になりました。

    • Parallelized hash joins and parallelized sequential scans now perform better パラレルハッシュ結合とパラレルシーケンシャルの実行が改善しました。

  • SQL stored procedures that support embedded transactions 組み込みトランザクションに対応したSQLストアドプロシージャ。

  • Optional Just-in-Time (JIT) compilation for some SQL code, speeding evaluation of expressions オプションで一部SQLコードに対するJist-in-Time(JIT)コンパイルに対応し、式の評価が高速化します。

  • Window functions now support all framing options shown in the SQL:2011 standard, including <literal>RANGE <replaceable>distance</replaceable> PRECEDING/FOLLOWING</literal>, <literal>GROUPS</literal> mode, and frame exclusion options ウィンドウ関数が、RANGE 距離 PRECEDING/FOLLOWING、GROUPS モード、およびフレームの除外オプションを含む、SQL:2011標準で示される全てのフレームオプションに対応するようになりました。

  • Covering indexes can now be created, using the <literal>INCLUDE</literal> clause of <command>CREATE INDEX</command> CREATE INDEXのINCLUDE句を使って、カバリングインデックスが作れるようになりました。

  • Many other useful performance improvements, including the ability to avoid a table rewrite for <command>ALTER TABLE ... ADD COLUMN</command> with a non-null column default 非NULLの列デフォルトを伴うALTER TABLE ... ADD COLUMNでテーブル書き直しを回避できることを含む、その他多数の有用な性能改善。

The above items are explained in more detail in the sections below. 上記の項目は次節でより詳しく説明されます。
