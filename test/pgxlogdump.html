O.3. pg_xlogdumpの名前をpg_waldumpに変更

<title><command>pg_xlogdump</command> renamed to <command>pg_waldump</command></title>

PostgreSQL 9.6 and below provided a command named <command>pg_xlogdump</command> <indexterm><primary>pg_xlogdump</primary></indexterm> to read write-ahead-log (WAL) files. This command was renamed to <command>pg_waldump</command>, see <xref linkend="pgwaldump"/> for documentation of <command>pg_waldump</command> and see <link linkend="release-prior">the release notes for PostgreSQL 10</link> for details on this change. PostgreSQL 9.6以下では、pg_xlogdumpという名前のコマンドが write-ahead-log(WAL)ファイルを読み取るために提供されていました。 このコマンドはpg_waldumpに名前が変更されました。 pg_waldumpのドキュメントについてはpg_waldumpを、この変更の詳細についてはPostgreSQL 10のリリースノートを参照してください。
