E.276. リリース8.0.4

<title>Release 8.0.4</title>

<title>Release date:</title> リリース日: 2005-10-04

This release contains a variety of fixes from 8.0.3. For information about new features in the 8.0 major release, see <xref linkend="release-8-0"/>. このリリースは8.0.3の各種不具合を修正したものです。 8.0メジャーリリースにおける新機能についてはE.280を参照してください。

E.276.1. バージョン8.0.4への移行

<title>Migration to Version 8.0.4</title>

A dump/restore is not required for those running 8.0.X. However, if you are upgrading from a version earlier than 8.0.3, see <xref linkend="release-8-0-3"/>. 8.0.Xからの移行ではダンプ/リストアは不要です。 しかしながら、8.0.3より以前のバージョンからアップグレードする場合は、E.277を参照してください。

E.276.2. 変更点

<title>Changes</title>
<listitem><para>Fix error that allowed <command>VACUUM</command> to remove <literal>ctid</literal> chains too soon, and add more checking in code that follows <literal>ctid</literal> links</para>

  • VACUUMによってctidチェーンがすぐに削除されてしまうエラーを修正し、ctidリンクを追跡するコードにおいてより多くの検査を行うようにしました。

    <para>This fixes a long-standing problem that could cause crashes in very rare circumstances.</para></listitem>

    これにより、長い間存在した、非常に稀な環境におけるクラッシュという問題が修正されました。

    • <listitem><para>Fix <type>CHAR()</type> to properly pad spaces to the specified length when using a multiple-byte character set (Yoshiyuki Asaba)</para>

  • マルチバイト文字セットを使用する場合に、CHAR()の空白文字埋め込みが適切に指定した長さになるように修正しました。(Yoshiyuki Asaba)

    <para>In prior releases, the padding of <type>CHAR()</type> was incorrect because it only padded to the specified number of bytes without considering how many characters were stored.</para></listitem>

    以前のリリースでは、CHAR()の空白文字埋め込みが正しくありませんでした。 格納する文字数を考慮せずに、単に指定されたバイト数で空白文字を埋め込んでいただけでした。

    • <listitem><para>Force a checkpoint before committing <command>CREATE DATABASE</command></para>

  • CREATE DATABASEをコミットする前にチェックポイントを強制実行します。

    <para>This should fix recent reports of <quote>index is not a btree</quote> failures when a crash occurs shortly after <command>CREATE DATABASE</command>.</para></listitem>

    これにより、CREATE DATABASEの実行後すぐにクラッシュしてしまった場合に起きる「index is not a btree」という失敗が修正されたはずです。

    • <listitem><para>Fix the sense of the test for read-only transaction in <command>COPY</command></para>

  • COPYにおける読みとりのみのトランザクションに関する試験の意味を修正しました。

    <para>The code formerly prohibited <command>COPY TO</command>, where it should prohibit <command>COPY FROM</command>.

    このコードでCOPY TOを禁止していましたが、本来ならばCOPY FROMを禁止しなければなりません。

    • <listitem><para>Handle consecutive embedded newlines in <command>COPY</command> CSV-mode input</para></listitem>

  • CSVモード入力のCOPYにおいて、連続して埋め込まれた改行を取り扱うようになりました。

    • <listitem><para>Fix <function>date_trunc(week)</function> for dates near year end</para></listitem>

  • 年末近くの日付に対するdate_trunc(week)を修正しました。

    • <listitem><para>Fix planning problem with outer-join ON clauses that reference only the inner-side relation</para></listitem>

  • リレーションの内部のみを参照する、外部結合のON句に関する計画作成の問題を修正しました。

    • <listitem><para>Further fixes for <literal>x FULL JOIN y ON true</literal> corner cases</para></listitem>

  • x FULL JOIN y ON trueの境界に関して更に改修しました。

    • <listitem><para>Fix overenthusiastic optimization of <literal>x IN (SELECT DISTINCT ...)</literal> and related cases</para></listitem>

  • x IN (SELECT DISTINCT ...)と関連する状態に対する過度な最適化を修正しました。

    • <listitem><para>Fix mis-planning of queries with small <literal>LIMIT</literal> values due to poorly thought out <quote>fuzzy</quote> cost comparison</para></listitem>

  • 「あいまいな」コスト比較の考慮が十分でなかったため、小さなLIMIT値の問い合わせに対する計画作成の間違いを修正しました。

    • <listitem><para>Make <function>array_in</function> and <function>array_recv</function> more paranoid about validating their OID parameter</para></listitem>

  • array_inとarray_recvのOIDパラメータの検証をより偏執的に行うようにしました。

    • <listitem><para>Fix missing rows in queries like <literal>UPDATE a=... WHERE a...</literal> with GiST index on column <literal>a</literal></para></listitem>

  • a列にGiSTインデックスを持つ場合のUPDATE a=... WHERE a...といった問い合わせにおいて行が見つからない点を修正しました。

    • <listitem><para>Improve robustness of datetime parsing</para></listitem>

  • 日付時刻の解析の堅牢性を向上しました。

    • <listitem><para>Improve checking for partially-written WAL pages</para></listitem>

  • 部分的に書き込まれたWALページに対する検査を改良しました。

    • <listitem><para>Improve robustness of signal handling when SSL is enabled</para></listitem>

  • SSL有効時のシグナルの扱いに対する堅牢性を向上しました。

    • <listitem><para>Improve MIPS and M68K spinlock code</para></listitem>

  • MIPSおよびM68Kのスピンロックコードを改良しました。

    • <listitem><para>Don't try to open more than <literal>max_files_per_process</literal> files during postmaster startup</para></listitem>

  • postmasterの起動時に、max_files_per_process以上のファイルを開かないようにしました。

    • <listitem><para>Various memory leakage fixes</para></listitem>

  • さまざまなメモリリークを修正しました。

    • <listitem><para>Various portability improvements</para></listitem>

  • 移植性の向上をいろいろと行いました。

    • <listitem><para>Update timezone data files</para></listitem>

  • タイムゾーンデータファイルを更新しました。

    • <listitem><para>Improve handling of DLL load failures on Windows</para></listitem>

  • WindowsにおけるDLLロード失敗の扱いを改良しました。

    • <listitem><para>Improve random-number generation on Windows</para></listitem>

  • Windowsにおける乱数発生を改良しました。

    • <listitem><para>Make <literal>psql -f filename</literal> return a nonzero exit code when opening the file fails</para></listitem>

  • psql -f filenameにおいて、指定したファイルを開くことができなかった場合の終了コードを非0にしました。

    • <listitem><para>Change <application>pg_dump</application> to handle inherited check constraints more reliably</para></listitem>

  • 継承された検査制約をより正確に扱うようにpg_dumpを変更しました。

    • <listitem><para>Fix password prompting in <application>pg_restore</application> on Windows</para></listitem>

  • Windowsにおいて、pg_restoreのパスワード入力待ちを修正しました。

    • <listitem><para>Fix PL/pgSQL to handle <literal>var := var</literal> correctly when the variable is of pass-by-reference type</para></listitem>

  • PL/pgSQLにおいて、変数が参照渡し型の場合のvar := varの扱いを修正しました。

    • <listitem><para>Fix PL/Perl <literal>%_SHARED</literal> so it's actually shared</para></listitem>

  • PL/Perlの%_SHAREDを修正し、本当に共有できるようになりました。

    • <listitem><para>Fix <filename>contrib/pg_autovacuum</filename> to allow sleep intervals over 2000 sec</para></listitem>

  • 2000秒以上の間隔で待機できるようにcontrib/pg_autovacuumを修正しました。

    • <listitem><para>Update <filename>contrib/tsearch2</filename> to use current Snowball code</para></listitem>

  • 正確なSnowballコードを使用するようにcontrib/tsearch2を更新しました。
