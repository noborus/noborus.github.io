E.136. リリース9.0.22

<title>Release 9.0.22</title>

<title>Release date:</title> リリース日: 2015-06-12

This release contains a small number of fixes from 9.0.21. For information about new features in the 9.0 major release, see <xref linkend="release-9-0"/>. このリリースは9.0.21に対し、少数の不具合を修正したものです。 9.0メジャーリリースにおける新機能については、E.158 を参照してください。

The <productname>PostgreSQL</productname> community will stop releasing updates for the 9.0.X release series in September 2015. Users are encouraged to update to a newer release branch soon. PostgreSQLコミュニティは2015年9月に9.0.Xシリーズの更新リリースを終了する予定です。 早めに新しいリリースのブランチに更新することを推奨します。

E.136.1. バージョン 9.0.22への移行

<title>Migration to Version 9.0.22</title>

A dump/restore is not required for those running 9.0.X. 9.0.Xからの移行ではダンプ/リストアは不要です。

However, if you are upgrading from a version earlier than 9.0.18, see <xref linkend="release-9-0-18"/>. しかしながら、9.0.18よりも前のリリースからアップグレードする場合は、E.140を参照して下さい。

E.136.2. 変更点

<title>Changes</title>

  • Fix rare failure to invalidate relation cache init file (Tom Lane) 稀にリレーションキャッシュ初期化ファイル無効化に失敗するのを修正しました。 (Tom Lane)

    With just the wrong timing of concurrent activity, a <command>VACUUM FULL</command> on a system catalog might fail to update the <quote>init file</quote> that's used to avoid cache-loading work for new sessions. This would result in later sessions being unable to access that catalog at all. This is a very ancient bug, but it's so hard to trigger that no reproducible case had been seen until recently. ちょうど悪いタイミングの同時動作で、システムカタログのVACUUM FULLが、新たなセッションのためにキャッシュ読み込み動作を避けるのに使われる「initファイル」の更新に失敗することがありました。 この結果、後のセッションがそのシステムカタログに全くアクセスできなくなってしまいます。 これはとても古くからのバグですが、起こすのが難しく、最近まで再現できるケースが見つかりませんでした。

  • Avoid deadlock between incoming sessions and <literal>CREATE/DROP DATABASE</literal> (Tom Lane) 新たなセッション開始とCREATE/DROP DATABASEとの間のデッドロックを回避しました。 (Tom Lane)

    A new session starting in a database that is the target of a <command>DROP DATABASE</command> command, or is the template for a <command>CREATE DATABASE</command> command, could cause the command to wait for five seconds and then fail, even if the new session would have exited before that. DROP DATABASEコマンドの対象であるか、または、CREATE DATABASEコマンドでのテンプレートであるデータベースに対する新たなセッション開始が、5秒待った後、たとえ新たなセッションがその前に終了していたとしても、失敗する可能性がありました。
