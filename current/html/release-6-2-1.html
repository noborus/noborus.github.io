E.361. リリース6.2.1

<title>Release 6.2.1</title>

<title>Release date:</title> リリース日: 1997-10-17

6.2.1 is a bug-fix and usability release on 6.2. 6.2.1 はリリース6.2 の不具合の改修と利便性を上げたものです。

Summary: 要約:

  • Allow strings to span lines, per <acronym>SQL92</acronym>. SQL92に従い、文字列を複数行に跨った形にできます。

  • Include example trigger function for inserting user names on table updates. テーブル更新時にユーザ名を挿入する、トリガ関数の例を含めました。

This is a minor bug-fix release on 6.2. For upgrades from pre-6.2 systems, a full dump/reload is required. Refer to the 6.2 release notes for instructions. これは 6.2 の小さな不具合を改修したものです。6.2 よりも前のシステムからアップグレードを行うためには、完全なダンプ/リロードが必要です。手順については 6.2 のリリースノートを参照して下さい。

E.361.1. バージョン6.2からバージョン6.2.1への移行

<title>Migration from version 6.2 to version 6.2.1</title>

This is a minor bug-fix release. A dump/reload is not required from version 6.2, but is required from any release prior to 6.2. これは小さな不具合を改修したリリースです。 バージョン6.2からの場合は、ダンプ/リロードは必要ありませんが、それ以前のリリースでは必要です。

In upgrading from version 6.2, if you choose to dump/reload you will find that avg(money) is now calculated correctly. All other bug fixes take effect upon updating the executables. バージョン6.2からのアップグレードで、ダンプ/リロードを行った場合、avg(money) の計算が正確になったことが判るでしょう。 この他全ての不具合の改修は、実行形式ファイルを更新した時点で効果が得られます。

Another way to avoid dump/reload is to use the following SQL command from <command>psql</command> to update the existing system table: ダンプ/リロードを行わない他の方法として、以下の SQL コマンドをpsql から実行して、既存のシステムテーブルを更新する方法があります。 

update pg_aggregate set aggfinalfn = 'cash_div_flt8'
 where aggname = 'avg' and aggbasetype = 790;

This will need to be done to every existing database, including template1. これは、template1 を含め、全ての既存データベースに対して行う必要があります。
