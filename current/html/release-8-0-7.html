E.273. リリース8.0.7

<title>Release 8.0.7</title>

<title>Release date:</title> リリース日: 2006-02-14

This release contains a variety of fixes from 8.0.6. For information about new features in the 8.0 major release, see <xref linkend="release-8-0"/>. このリリースは8.0.6の各種不具合を修正したものです。 8.0メジャーリリースにおける新機能についてはE.280を参照してください。

E.273.1. バージョン8.0.7への移行

<title>Migration to Version 8.0.7</title>

A dump/restore is not required for those running 8.0.X. However, if you are upgrading from a version earlier than 8.0.6, see <xref linkend="release-8-0-6"/>. 8.0.Xからの移行ではダンプ/リストアは不要です。 しかしながら、8.0.6より前のバージョンからアップグレードする場合は、E.274を参照してください。

E.273.2. 変更点

<title>Changes</title>
<listitem><para>Fix potential crash in <command>SET SESSION AUTHORIZATION</command> (CVE-2006-0553)</para>

  • SET SESSION AUTHORIZATIONによるクラッシュの可能性を修正しました。 (CVE-2006-0553)

    <para>An unprivileged user could crash the server process, resulting in momentary denial of service to other users, if the server has been compiled with Asserts enabled (which is not the default). Thanks to Akio Ishida for reporting this problem.

    アサートを有効にしてコンパイルした場合（これはデフォルトではありません）、非特権ユーザがサーバをクラッシュでき、その結果、他のユーザへのサービスが一時的に不能になります。 この問題を報告してくれたAkio Ishidaに感謝します。

    • <listitem><para>Fix bug with row visibility logic in self-inserted rows (Tom)</para>

  • 自身が挿入した行に関する行の可視性ロジックについての不具合を修正しました。 (Tom)

    <para>Under rare circumstances a row inserted by the current command could be seen as already valid, when it should not be. Repairs bug created in 8.0.4, 7.4.9, and 7.3.11 releases.

    非常に稀な状況で、現在のコマンドが挿入した行が、本来無効であるべき時に既に有効になっているものとして扱われることがありました。 リリース8.0.4、7.4.9、7.3.11で組み込まれた不具合を修正しました。

    • <listitem><para>Fix race condition that could lead to <quote>file already exists</quote> errors during pg_clog and pg_subtrans file creation

  • pg_clogおよびpg_subtransファイル生成時に「ファイルが既に存在する」というエラーを引き起こす競合条件を修正しました。 (Tom)

    • <listitem><para>Fix cases that could lead to crashes if a cache-invalidation message arrives at just the wrong time (Tom)</para></listitem>

  • キャッシュ無効化メッセージがおかしな時刻に届いた場合にクラッシュを引き起こす可能性を修正しました。 (Tom)

    • <listitem><para>Properly check <literal>DOMAIN</literal> constraints for <literal>UNKNOWN</literal> parameters in prepared statements

  • 準備済み文内のUNKNOWNパラメータ用のDOMAIN制約を適切に検査します。 (Neil)

    • <listitem><para>Ensure <command>ALTER COLUMN TYPE</command> will process <literal>FOREIGN KEY</literal>, <literal>UNIQUE</literal>, and <literal>PRIMARY KEY</literal> constraints in the proper order (Nakano Yoshihisa)</para></listitem>

  • ALTER COLUMN TYPEがFOREIGN KEY、UNIQUE、PRIMARY KEY制約を確実に適切な順番で処理するようにしました。 (Nakano Yoshihisa)

    • <listitem><para>Fixes to allow restoring dumps that have cross-schema references to custom operators or operator classes (Tom)</para></listitem>

  • 独自の演算子や演算子クラスを参照するクロススキーマを持つダンプをリストアできるように修正しました。 (Tom)

    • <listitem><para>Allow <application>pg_restore</application> to continue properly after a <command>COPY</command> failure; formerly it tried to treat the remaining <command>COPY</command> data as SQL commands (Stephen Frost)</para></listitem>

  • COPYに失敗した後、pg_restoreが適切に処理を継続できるようにしました。 これまではCOPYデータの残りをSQLコマンドとして扱おうとしました。 (Stephen Frost)

    • <listitem><para>Fix <application>pg_ctl</application> <literal>unregister</literal> crash when the data directory is not specified (Magnus)</para></listitem>

  • データディレクトリを指定しない場合にpg_ctl unregisterがクラッシュすることを修正しました。 (Magnus)

    • <listitem><para>Fix <application>ecpg</application> crash on AMD64 and PPC

  • libpq PQprint HTMLタグを修正しました。 (Neil)

    • <listitem><para>Recover properly if error occurs during argument passing in <application>PL/Python</application> (Neil)</para></listitem>

  • PL/Pythonにおいて、引数を渡す段階でエラーが発生した場合に適切に復旧するようにしました。 (Neil)

    • <listitem><para>Fix <application>PL/Perl</application>'s handling of locales on Win32 to match the backend (Andrew)</para></listitem>

  • Win32において、PL/Perlのロケールの扱いをバックエンドに合わせるように修正しました。 (Andrew)

    • <listitem><para>Fix crash when <literal>log_min_messages</literal> is set to <literal>DEBUG3</literal> or above in <filename>postgresql.conf</filename> on Win32

  • Win32においてpostgresql.confでlog_min_messagesがDEBUG3以上に設定された時のクラッシュを修正しました。 (Bruce)

    • <listitem><para>Fix <application>pgxs</application> <literal>-L</literal> library path specification for Win32, Cygwin, macOS, AIX (Bruce)</para></listitem>

  • Win32、Cygwin、macOS、AIX用のpgxs -Lライブラリ検索パス指定を修正しました。 (Bruce)

    • <listitem><para>Check that SID is enabled while checking for Win32 admin privileges (Magnus)</para></listitem>

  • Win32の管理者権限を検査する時にSIDが有効か検査します。 (Magnus)

    • <listitem><para>Properly reject out-of-range date inputs (Kris Jurka)</para></listitem>

  • 範囲外の日付入力を適切に拒否します。 (Kris Jurka)

    • <listitem><para>Portability fix for testing presence of <function>finite</function> and <function>isinf</function> during configure (Tom)</para></listitem>

  • configure時のfiniteおよびisinfの存在検査に関する移植性を修正しました。 (Tom)
