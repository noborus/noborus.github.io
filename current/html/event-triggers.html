<title>Event Triggers</title>

To supplement the trigger mechanism discussed in <xref linkend="triggers"/>, <productname>PostgreSQL</productname> also provides event triggers. Unlike regular triggers, which are attached to a single table and capture only DML events, event triggers are global to a particular database and are capable of capturing DDL events. 第39章で議論されたトリガの機能を補完するために、PostgreSQLはイベントトリガを提供します。 一つのテーブルに付与され、DMLイベントのみ対象にしたこれまでのトリガと違い、イベントトリガは特定のデータベースに大域的であり、DDLイベントを対象に実行できます。

Like regular triggers, event triggers can be written in any procedural language that includes event trigger support, or in C, but not in plain SQL. これまでのトリガと違い、イベントトリガは普通のSQLではなく、イベントトリガがサポートする手続き言語やC言語で記述することができます。
