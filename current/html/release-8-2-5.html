E.224. リリース8.2.5

<title>Release 8.2.5</title>

<title>Release date:</title> リリース日: 2007-09-17

This release contains a variety of fixes from 8.2.4. For information about new features in the 8.2 major release, see <xref linkend="release-8-2"/>. このリリースは8.2.4の各種不具合を修正したものです。 8.2メジャーリリースにおける新機能についてはE.229を参照してください。

E.224.1. バージョン8.2.5への移行

<title>Migration to Version 8.2.5</title>

A dump/restore is not required for those running 8.2.X. 8.2.Xからの移行ではダンプ/リストアは不要です。

E.224.2. 変更点

<title>Changes</title>

  • Prevent index corruption when a transaction inserts rows and then aborts close to the end of a concurrent <command>VACUUM</command> on the same table (Tom) トランザクションが、行を挿入した後に、同一テーブルに対する同時実行中のVACUUMの終了時刻とほぼ同じ時点でアボートした場合のインデックス破損を防止します。(Tom)

  • Fix <literal>ALTER DOMAIN ADD CONSTRAINT</literal> for cases involving domains over domains (Tom) ドメイン越しのドメインを含む場合のALTER DOMAIN ADD CONSTRAINTを修正しました。(Tom)

  • Make <command>CREATE DOMAIN ... DEFAULT NULL</command> work properly (Tom) CREATE DOMAIN ... DEFAULT NULLが適切に動作するようにしました。(Tom)

  • Fix some planner problems with outer joins, notably poor size estimation for <literal>t1 LEFT JOIN t2 WHERE t2.col IS NULL</literal> (Tom) 外部結合に関するプランナの問題をいくつか、特に、t1 LEFT JOIN t2 WHERE t2.col IS NULLに対するおかしなサイズ推定を修正しました。

  • Allow the <type>interval</type> data type to accept input consisting only of milliseconds or microseconds (Neil) intervalデータ型が、ミリ秒のみ、または、マイクロ秒のみから構成された入力を受け付けられるようになりました。(Neil)

  • Allow timezone name to appear before the year in <type>timestamp</type> input (Tom) timestamp型の入力で、時間帯名称が年の前に来てもよくなりました。(Tom)

  • Fixes for <acronym>GIN</acronym> indexes used by <filename>/contrib/tsearch2</filename> (Teodor) /contrib/tsearch2で使用されるGINインデックスを修正しました。(Teodor)

  • Speed up rtree index insertion (Teodor) r-treeインデックスの挿入を高速化しました。(Teodor)

  • Fix excessive logging of <acronym>SSL</acronym> error messages (Tom) 過度なSSLエラーメッセージのログ処理を修正しました。(Tom)

  • Fix logging so that log messages are never interleaved when using the syslogger process (Andrew) sysloggerプロセスを使用する時にログメッセージが差し込まれないようにログ処理を修正しました。(Andrew)

  • Fix crash when <varname>log_min_error_statement</varname> logging runs out of memory (Tom) log_min_error_statementログ処理がメモリ不足になった時のクラッシュを修正しました。(Tom)

  • Fix incorrect handling of some foreign-key corner cases (Tom) 一部の外部キーの境界の不正な扱いを修正しました。(Tom)

  • Fix <function>stddev_pop(numeric)</function> and <function>var_pop(numeric)</function> (Tom) stddev_pop(numeric)とvar_pop(numeric)を修正しました。(Tom)

  • Prevent <command>REINDEX</command> and <command>CLUSTER</command> from failing due to attempting to process temporary tables of other sessions (Alvaro) 他のセッションによる一時テーブル処理を防ぐために起こるREINDEXとCLUSTERの失敗を防ぎます。(Alvaro)

  • Update the time zone database rules, particularly New Zealand's upcoming changes (Tom) 時間帯データベース規則を更新しました。具体的にはニュージーランドで予定している変更です。(Tom)

  • Windows socket and semaphore improvements (Magnus) Windowsソケットとセマフォを改良しました。(Magnus)

  • Make <command>pg_ctl -w</command> work properly in Windows service mode (Dave Page) pg_ctl -wがWin32サービスモードで適切に動作するようになりました。(Dave Page)

  • Fix memory allocation bug when using <application>MIT Kerberos</application> on Windows (Magnus) WindowsでMIT Kerberosを使用する場合のメモリ割り当てに関する不具合を修正しました。(Magnus)

  • Suppress timezone name (<literal>%Z</literal>) in log timestamps on Windows because of possible encoding mismatches (Tom) 符号化方式が不整合となる可能性があるため、Windowsにてログのタイムスタンプに時間帯名(%Z)を出力しないようにしました。(Tom)

  • Require non-superusers who use <filename>/contrib/dblink</filename> to use only password authentication, as a security measure (Joe) セキュリティ強化のため、パスワード認証のみを使用する/contrib/dblinkを使用するユーザが非特権ユーザであることを要求します。(Joe)

  • Restrict <filename>/contrib/pgstattuple</filename> functions to superusers, for security reasons (Tom) セキュリティ上の理由で、/contrib/pgstattuple関数をスーパーユーザに制限しました。(Tom)

  • Do not let <filename>/contrib/intarray</filename> try to make its GIN opclass the default (this caused problems at dump/restore) (Tom) /contrib/intarrayがGIN演算子クラスをデフォルトにしないようにしました（ダンプ/リストア時に問題が発生します）。(Tom)
