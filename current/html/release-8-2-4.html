E.225. リリース8.2.4

<title>Release 8.2.4</title>

<title>Release date:</title> リリース日: 2007-04-23

This release contains a variety of fixes from 8.2.3, including a security fix. For information about new features in the 8.2 major release, see <xref linkend="release-8-2"/>. このリリースは8.2.3の各種不具合を修正したもので、セキュリティ問題の修正を1つ含みます。 8.2メジャーリリースにおける新機能についてはE.229を参照してください。

E.225.1. バージョン8.2.4への移行

<title>Migration to Version 8.2.4</title>

A dump/restore is not required for those running 8.2.X. 8.2.Xからの移行ではダンプ/リストアは不要です。

E.225.2. 変更点

<title>Changes</title>

  • Support explicit placement of the temporary-table schema within <varname>search_path</varname>, and disable searching it for functions and operators (Tom) search_pathにおける一時テーブルスキーマの明示的な指定をサポートしました。 また、関数や演算子ではその検索を無効にしました(Tom)。

    This is needed to allow a security-definer function to set a truly secure value of <varname>search_path</varname>. Without it, an unprivileged SQL user can use temporary objects to execute code with the privileges of the security-definer function (CVE-2007-2138). See <command>CREATE FUNCTION</command> for more information. これは、SECURITY DEFINER関数がsearch_pathの値を本当に安全に設定するために必要です。 こうしないと、一時オブジェクトを使用して権限のないSQLユーザがSECURITY DEFINER関数の権限でコードを実行することができてしまうためです（CVE-2007-2138）。 詳細はCREATE FUNCTIONを参照してください。

  • Fix <varname>shared_preload_libraries</varname> for Windows by forcing reload in each backend (Korry Douglas) Windowsにおいて、各バックエンドで再ロードを強制的に行うことでshared_preload_librariesを修正しました（Korry Douglas）。

  • Fix <function>to_char()</function> so it properly upper/lower cases localized day or month names (Pavel Stehule) 翻訳された日や月名を適切に大文字、小文字に変換できるように、to_char()を修正しました。

  • <filename>/contrib/tsearch2</filename> crash fixes (Teodor) /contrib/tsearch2のクラッシュを修正しました(Teodor)。

  • Require <command>COMMIT PREPARED</command> to be executed in the same database as the transaction was prepared in (Heikki) COMMIT PREPAREDは、そのトランザクションを準備したデータベースと同じデータベースで実行する必要があります。(Heikki)

  • Allow <command>pg_dump</command> to do binary backups larger than two gigabytes on Windows (Magnus) Windowsにおいて、2ギガバイトを越えるバイナリバックアップをpg_dumpが行えるようになりました(Magnus)。

  • New traditional (Taiwan) Chinese <acronym>FAQ</acronym> (Zhou Daojing) 伝統的中国（台湾）語のFAQを更新しました(Zhou Daojing)。

  • Prevent the statistics collector from writing to disk too frequently (Tom) 統計情報コレクタがあまりに頻繁にディスクに書き込まないようにしました(Tom)。

  • Fix potential-data-corruption bug in how <command>VACUUM FULL</command> handles <command>UPDATE</command> chains (Tom, Pavan Deolasee) VACUUM FULLのUPDATE連鎖方法において潜在したデータ破損不具合を修正しました(Tom, Pavan Deolasee)。

  • Fix bug in domains that use array types (Tom) 配列型を使用するドメインにおける不具合を修正しました(Tom)。

  • Fix <command>pg_dump</command> so it can dump a serial column's sequence using <option>-t</option> when not also dumping the owning table 元となるテーブルのダンプを行わない場合でも、-tを使用して連番型の列のシーケンスをダンプできるようpg_dumpを修正しました。 (Tom)

  • Planner fixes, including improving outer join and bitmap scan selection logic (Tom) 外部結合の向上やビットマップスキャンの選択ロジックなど、プランナを修正しました(Tom)。

  • Fix possible wrong answers or crash when a PL/pgSQL function tries to <literal>RETURN</literal> from within an <literal>EXCEPTION</literal> block PL/pgSQL関数がEXCEPTIONブロック内でRETURNを試行する場合に潜在する誤作動やクラッシュの可能性を修正しました。 (Tom)

  • Fix PANIC during enlargement of a hash index (Tom) ハッシュインデックスの拡張時のパニックを修正しました(Tom)。

  • Fix POSIX-style timezone specs to follow new USA DST rules (Tom) POSIX書式の時間帯指定が新しいUSA DST規則に従うよう修正しました(Tom)。
